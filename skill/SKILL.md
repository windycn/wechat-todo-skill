# WeChat Todo Skill

## 触发场景

用户请求涉及以下任一场景时触发：
- 回顾微信聊天内容
- 提取聊天中的待办事项
- 总结群聊/单聊的关键信息
- 查找特定时间段的微信对话
- 分析微信聊天中的截止日期

## 交互流程

### 第一步：确认数据库

系统会自动查找微信数据库。如果未找到，会显示详细的数据库准备步骤：

**权限要求**：
- **Windows 用户**：请确保以管理员权限启动支持 skills 的 agent 程序，否则可能无法从微信内存中提取密钥。
- **macOS 用户**：由于系统安全限制（SIP），无法像 Windows 那样直接从内存提取密钥。需要手动准备解密数据，具体步骤如下：
  * **步骤1**: 定位微信数据目录
    - 打开 Finder，按 `Cmd+Shift+G` 输入路径：`~/Library/Containers/com.tencent.xinWeChat/Data/Library/Application Support/com.tencent.xinWeChat/`
    - 进入 `4.0b*/` 目录（`*` 代表一串随机字符），找到 `db_storage` 文件夹
  * **步骤2**: 使用第三方工具提取并解密数据库
    - 推荐使用开源工具 `https://github.com/ylytdeng/wechat-decrypt` 或其他类似的微信数据库解密工具
    - 按照工具说明提取并解密微信数据库文件
  * **步骤3**: 将解密后的数据复制到用户对话路径下
    - 将解密工具生成的 `decrypted` 目录完整复制到当前用户对话路径下
    - 确保目录结构为：`{用户对话路径}/decrypted/message_*.db`

**提示**：如果电脑端微信缺少数据，请使用微信的手机聊天记录导出功能导出到电脑微信。

### 第二步：确认分析对象

询问用户要分析的对象：

**若用户未指定**，询问：
> "你想分析哪个群或好友的聊天记录？请提供群名、昵称、备注名或微信号。"

**若用户已指定**，直接进入下一步。

### 第三步：处理模糊匹配

如果用户输入匹配到多个结果，必须列举让用户确认：
> "找到多个匹配项，请选择你要分析的对象：
>  1. [群名/昵称] (备注: xxx)
>  2. [群名/昵称] (备注: xxx)
>  请输入编号或名称进行确认。"

### 第四步：确认日期范围

提供预设选项：
> "请选择要分析的时间范围：
>  - 1) 今日
>  - 2) 昨日
>  - 3) 本周
>  - 4) 最近7天
>  - 5) 自定义范围
>  请输入选项编号。"

用户选择"自定义范围"时，进一步询问具体日期。

### 第五步：提供预设方案

可主动推荐快捷方案：
> "我可以提供以下快速分析：
>  - 今日工作汇总：今天内的待办事项 + DDL
>  - 本周回顾：本週完整摘要 + 关键事件
>  - 待办提取：所有待办事项
>  需要使用预设方案吗？"

## 输出格式

分析完成后，输出结构化报告：

```
## 聊天摘要
- 会话对象: [群名/好友名称]
- 时间范围: [日期起止]
- 消息总数: [N条]
- 分析结果: [主要讨论内容]

## 待办事项
1. [待办内容] (来源: [发送者] @ [时间])
2. [待办内容] (来源: [发送者] @ [时间])
...

## 截止日期 (DDL)
1. [时间点] - [相关上下文]
2. ...

## 关键事件
1. [事件描述] (@ [时间])
2. ...
```

若无待办/DDL/事件，明确告知"未提取到相关事项"。

## 技术规范

- 数据库必须已解密
- 联系人匹配优先级：精确匹配备注名 > 精确匹配昵称 > 精确匹配微信号 > 模糊匹配
- 时间范围使用中国时区 (UTC+8)
- 所有数据处理在本地完成，不涉及网络传输
- 支持增量分析，避免重复处理数据
- 高性能分析，处理速度可达350万条/秒
- 三级缓存机制，缓存查询速度可达1959万次/秒
- 自动数据库查找和解密，减少用户交互
- 支持Windows平台自动解密，macOS平台提供详细的手动解密步骤

## 处理规则

1. **用户未指定分析对象** → 询问确认
2. **匹配到多个结果** → 列举让用户选择
3. **数据库路径无效** → 提示错误并引导重新输入
4. **日期范围不明确** → 使用"今日"作为默认
5. **用户有其他需求** → 遵循用户意愿调整分析范围
6. **电脑端微信缺少数据** → 提醒用户使用手机聊天记录导出功能
